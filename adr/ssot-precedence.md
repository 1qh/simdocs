## Drift fences

- `MIPS-ISA.md` is spec-of-code per `book/HARD-RULES.md` allowlist; CI lint diffs the doc's opcode table against the generated `isa.ts` table.
- `MIPS-DATAPATH.md` is spec-of-code; CI lint diffs the doc's path / segment / value-id / control-signal tables against `topology.ts`.
- `STACK.md` is spec-of-code; CI lint diffs the picks against `package.json` deps + source imports.

## No hand-typed cross-repo strings

Per `book/HARD-RULES.md` "No code import across repos". Facts that flow `simdocs` → `sim` flow as generated artifacts (e.g. `apps/backend/convex/generated/isa.ts` generated by a build step that reads `simdocs/MIPS-ISA.md`). Hand-typing the opcode table into both = violation.

## Caught by

- Per-allowlist-entry CI lint (above).
- Manifest lint asserts every allowlist entry has a wired lint script.
