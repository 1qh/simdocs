# A11Y

Accessibility floor. WCAG AA minimum. World-class endpoint requires more.

## Contrast

- Body text vs background: ≥ 7:1 (AAA tier)
- UI chrome: ≥ 4.5:1 (AA)
- Active accent: ≥ 4.5:1 against background
- Hazard amber: ≥ 4.5:1
- Disabled elements: ≥ 3:1 (must still be perceivable)

Color is NEVER the sole carrier of meaning. Hazards = amber + icon + label. Active components = accent emissive + thicker outline + tooltip.

## Color-blind palettes

Three alternate palettes selectable in settings (locked accent shifts):
- **Deuteranopia** — accent shifts cyan → blue-purple, hazard amber → magenta
- **Protanopia** — accent shifts cyan → indigo, hazard amber → bright-yellow
- **Tritanopia** — accent shifts cyan → red, hazard amber → green-shifted

Auto-detected via `prefers-color-scheme: <variant>` if browser ever standardizes; manual toggle until then.

## Screen readers

3D canvas is invisible to screen readers — DOM proxies carry the semantics:

| 3D entity | DOM proxy |
|---|---|
| ALU during EX step | `<button role="img" aria-label="ALU active, computing add of {op1} and {op2}, result {result}">` |
| Control unit signal | `<dt aria-label="RegDst signal value 1">RegDst</dt><dd>1</dd>` |
| Register file row | `<tr aria-label="register $t0, value 0x{value}">...</tr>` |
| Memory word | `<tr aria-label="address 0x{addr}, value 0x{value}">...</tr>` |
| K-map cell | `<button role="gridcell" aria-label="cell {minterm}, output {0\|1\|X}">` |
| K-map group | `<div role="region" aria-label="group covering minterms {list}, product term {expr}">` |
| Pipeline cell | `<button role="gridcell" aria-label="instruction {n}, cycle {c}, stage {s}">` |

Live regions announce step changes: `<div role="status" aria-live="polite">Step IF complete. PC = 0x{...}. Fetched instruction add $t0, $t1, $t2.</div>`

## Reduced motion

`prefers-reduced-motion: reduce` collapses every animation to instantaneous state swap:
- 3D camera dolly → instant cut to bookmark
- Signal pulse → static highlight (no traveling light)
- Stage transition focus pulls → instant focus change
- Pipeline diagram cell fades → no fade, immediate appearance
- K-map group rectangle → no draw animation, snap to final
- Step easing → no easing, immediate

Functional equivalent to motion mode; no information lost.

## Focus visible

Custom focus rings: 2px solid accent, 2px offset from element bevel. Visible on every focusable element. Never `outline: none` without replacement.

## Form labels

Every input has an explicit `<label for="">` or `aria-labelledby` pointing at a visible text element. Placeholder text never substitutes for label.

## ARIA practices

- `aria-busy` during sim step computation
- `aria-disabled` instead of `disabled` attribute where keyboard reach matters
- `aria-describedby` for complex 3D entities (longer description than `aria-label` allows)
- `role="application"` on the main 3D canvas region to switch screen-reader to "browse" mode less aggressively

## Caught by

- `eslint-plugin-jsx-a11y` rules all enabled
- Playwright a11y test: `axe-core` runs on every route, zero violations required
- Color contrast test: `pa11y` or equivalent runs against every theme variant
- Keyboard-only smoke: Playwright sets `keyboardOnly: true`, exercises every route's documented matrix
- Screen reader smoke: NVDA / VoiceOver scripted walk-throughs in CI (or manual checklist per release)
